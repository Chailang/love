## 优先级排序逻辑

按「用户能否跑通完整核心闭环」倒推：

| 优先级 | 定义 |
|--------|------|
| **P0** | 系统跑不起来，用户啥也做不了 |
| **P1** | 有 P0 但不能形成产品闭环 |
| **P2** | 闭环可用，体验加分项 |

---

## 🔴 P0 — 基础设施层

| # | 模块 | 说明 | 状态 |
|---|------|------|:--:|
| 1 | **项目脚手架搭建** | Spring Boot + Flutter 项目初始化、Docker Compose | ✅ |
| 2 | **用户认证体系** | 注册/登录（手机号+验证码）、实名认证、学历认证、JWT | ✅ |
| 3 | **个人主页（基础版）** | 头像/昵称/年龄/学历/职业/城市/身高/星座 CRUD | ✅ |

## 🟡 P1 — 核心产品闭环

| # | 模块 | 说明 | 状态 |
|---|------|------|:--:|
| 4 | **匹配推荐 —「寻觅」** | 每日推荐卡片流、喜欢/跳过、推荐算法 | ✅ |
| 5 | **双向匹配机制** | 喜欢记录、互相喜欢 → 匹配通知 | ✅ |
| 6 | **即时通讯 —「聊天」** | WebSocket 实时消息、文字/图片、已读状态 | ✅ |

## 🟢 P2 — 体验增强层

| # | 模块 | 说明 | 状态 |
|---|------|------|------|
| 7 | **个人主页（完整版）** | 相册、兴趣标签、个人简介、资料完整度 | ⬜ |
| 8 | **超级曝光** | 付费曝光订单、推荐加权、效果追踪 | ⬜ |
| 9 | **会员与付费** | VIP 订阅、增值道具（超级喜欢/超级消息） | ⬜ |

---

## 一期 MVP 上线标准

```
新用户流程：
  手机号注册 → 学历认证 → 完善基础资料
  → 收到每日推荐 → 滑动喜欢
  → 双方互相匹配 → 开始聊天 ✅
```

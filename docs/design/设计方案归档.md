### 1.2 Flutter 选型决策 (2026-06-09)

**变更**：从小程序框架从 uniapp 切换为 Flutter

**理由**：
- 缘分盲盒的卡片翻转/粒子动画、骰子滚动、扭蛋裂开特效、SSR 光效对动画要求极高
- Flutter 的 Skia 引擎提供 60fps 原生渲染
- 一套代码同时支持微信小程序和未来可能的其他端

**影响**：
- 前端团队需学习 Dart 语言
- 动画实现复杂度大幅降低（Flutter 内置动画库 vs uniapp CSS/JS）
- UI 一致性提升（像素级还原）

## 二、产品设计决策

### 2.1 产品定位

| 维度 | 决策 |
|------|------|
| **定位** | 高学历优质青年严肃婚恋交友平台 |
| **差异化** | 学历认证+实名认证双重信任壁垒 |
| **slogan** | 「在小镇之恋，遇见和你一样优秀的TA」 |
| **设计调性** | 温暖、走心、安全、可靠 |

### 2.2 新增三大功能 (2026-06-08)

| 功能 | 设计理念 | 核心玩法 |
|------|---------|---------|
| 🎲 **缘分盲盒** | 游戏化运气匹配，降低决策疲劳 | 盲盒/骰子/扭蛋三种玩法，SSR 稀有度系统 |
| 🔮 **八字合缘** | 传统文化赋能，趣味性匹配维度 | 万年历推算天干地支，合婚指数 0-100 分 |
| 📍 **同乡近邻** | 多维度地理匹配，提升线下见面率 | 家乡/工作/居住三维定位，距离模糊化隐私保护 |

### 2.3 付费体系设计

```
收入结构：
├── 会员订阅（核心收入）
├── 增值道具（超级喜欢/超级消息/超级曝光）
├── 缘分盲盒付费（盲盒¥5/次、骰子¥3/次、单抽¥3、十连¥25）
├── 八字 VIP 权益（详细合缘报告/筛选排序）
└── 同乡近邻 VIP（多维组合筛选/距离排序/精确距离）
```

### 2.4 匹配体系架构

```
用户画像
  ├── 基础信息（学历/职业/年薪/身高）   ← 原有
  ├── 八字信息（出生日期 → 天干地支）    ← 新增
  └── 地理信息（家乡/工作区域/住址区域） ← 新增
       ↓
  推荐算法
  ├── 智能推荐（原有）
  ├── 缘分盲盒（运气驱动）              ← 新增
  ├── 八字合缘（命理权重）              ← 新增
  └── 同乡近邻（地理权重）              ← 新增
```

### 5.2 关键算法决策

| 算法 | 核心逻辑 |
|------|---------|
| **八字合缘** | 年柱20% + 月柱25% + 日柱35% + 时柱20%，天干五合/地支六合三合/五行生克 |
| **扭蛋概率** | N:60% / R:25% / SR:12% / SSR:3%，30抽保底 SSR |
| **距离排序** | PostGIS 空间索引 (Geohash + R-tree) |
| **村口推荐** | 时间衰减30% + 互动热度40% + 兴趣匹配20% + 好友权重10% |

### 8.5 技术亮点

- **JWT 安全体系**：无状态认证 + 手机号验证码 + 学历/实名双重认证
- **WebSocket STOMP**：实时消息推送，按会话隔离，已读状态追踪
- **八字算法**：万年历天干地支推算 + 四柱加权评分（年20% 月25% 日35% 时20%）
- **Spring Security 6**：无状态会话 + 自定义 Filter 链 + 路径级权限控制
- **渐进式模块化**：包级隔离（user/match/chat/bazi），可独立拆微服务

## 九、Flutter 小程序技术方案

> 决策：2026-06-09 皇帝陛下决策，小程序从 uniapp 切换为 Flutter

### 9.1 为什么选 Flutter

| 维度 | Flutter | uniapp |
|------|---------|--------|
| **动画渲染** | Skia 引擎 60fps 原生渲染 | CSS/JS 动画，性能受限 |
| **盲盒特效** | 卡片翻转/粒子/扭蛋裂开/SSR 光效零压力 | 动画实现复杂，卡顿风险高 |
| **UI 一致性** | 像素级还原，设计稿即代码 | 不同端渲染差异大 |
| **多端支持** | 微信小程序（Flutter MP）+ iOS/Android | 仅小程序/H5 |
| **开发体验** | Dart 静态类型 + hot reload | JS 弱类型 |

### 9.2 Flutter 技术栈建议

```
小程序端：
├── flutter_wechat ← 微信小程序 SDK 集成（微信登录/支付/分享）
├── flutter_secure_storage ← 本地 JWT 安全存储
├── stomp_dart_client ← WebSocket 聊天
├── flutter_lottie ← 盲盒动画特效
├── flutter_animations ← 卡片翻转/粒子效果
└── cached_network_image ← 头像缓存优化
```
